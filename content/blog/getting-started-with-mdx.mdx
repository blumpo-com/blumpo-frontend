## What is MDX?

**MDX** is a format that lets you use JSX in your Markdown documents. You can import and use React components within your content, making it incredibly powerful for technical documentation and blogs.

## Why Use MDX?

### 1. Best of Both Worlds

You get the **simplicity of Markdown** for writing content and the **flexibility of React** for interactive elements.

### 2. Component Reusability

Write components once, use them across multiple posts:

```jsx
<Alert type="warning">
  This is an important warning!
</Alert>
```

### 3. No HTML Limitations

Unlike pure Markdown, you're not limited to basic HTML. You can create rich, interactive experiences.

## Using Components in MDX

Here's where MDX shines. You can import and use React components:

```mdx
import { CustomButton } from './components/Button'

# My Post

Here's some content with a custom button:

<CustomButton onClick={() => alert('Clicked!')}>
  Click Me
</CustomButton>
```

## Best Practices

### 1. Keep It Simple

Don't overuse components. Let Markdown do the heavy lifting for content.

### 2. Use TypeScript

Type your components for better DX:

```typescript
interface AlertProps {
  type: 'info' | 'warning' | 'error';
  children: React.ReactNode;
}

export function Alert({ type, children }: AlertProps) {
  // ...
}
```

### 3. Optimize Images

Always use Next.js `Image` component:

```jsx
import Image from 'next/image'
import heroImage from './hero.jpg'

<Image 
  src={heroImage} 
  alt="Hero" 
  width={800} 
  height={400}
/>
```

### 4. Test Your Components

Make sure components work in both MDX and regular React:

```tsx
// Can be used in MDX and .tsx files
export function CodeBlock({ children, language }) {
  return (
    <pre>
      <code className={`language-${language}`}>
        {children}
      </code>
    </pre>
  );
}
```

## Debugging MDX

Common issues and solutions:

### 1. JSX Syntax Errors

Make sure your JSX is properly formatted:

```mdx

```

### 2. Import Paths

Use relative paths for local components:

```mdx
import { Button } from '../components/Button'
```

### 3. Blank Lines

MDX can be picky about blank lines. When in doubt, add them:

```mdx
## Heading

Some text here.

<Component />

More text here.
```
